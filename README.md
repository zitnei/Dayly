📱 今日作ったAndroidアプリの説明
■ アプリ名

DailyOne / RhythmLife

■ アプリの概要

このアプリは、日の行動・状態・リズムを記録して、あとから見返すためのAndroidアプリです。

毎日の出来事や状態を残し、履歴として管理することで、
「自分が何をしてきたか」「どんな流れで生活しているか」を
アプリ上で振り返れる構成になっています。

カテゴリとしては、
ライフログ / デイリーログ / 記録・振り返り系アプリにあたります。

■ できること（構成から確実に言える内容）

このアプリは、画面と構造が次のように分かれています。

・Home画面

アプリを開いたときのメイン画面。
現在の状態や、最近の記録を確認するための入口画面です。

・Add画面

新しい記録を追加する画面。
その日の行動・状態・内容を入力して保存する役割を持ちます。

・Log画面

過去の記録を一覧で表示する画面。
これまでに残した内容を振り返るための履歴ページです。

■ 技術的な構成

このアプリは、Androidの標準的な実務構成で作られています。

・MainActivity
→ Androidアプリの起動入口
→ ComposeとアプリUIを接続する役割

・App構成（App.kt）
→ 画面遷移と全体構造の管理
→ Home / Add / Log 画面の切り替え

・ViewModel（MainViewModel）
→ アプリ内データの管理
→ 画面間で同じ情報を共有する中枢

・UI（screens）
→ 実際にユーザーが操作する画面群

つまり、
「表示」「ロジック」「状態管理」が分離された、実務向け構造になっています。
